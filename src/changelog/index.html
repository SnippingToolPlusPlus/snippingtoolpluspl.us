6.3.0 Latest Stable Release

New Features!

  • Customizeable Hotkey Support! - Now you can customize the hotkeys for your own personal needs.

General Improvements

  • Improved the Editor keybinds, they shouldn't be as finicky anymore and using CTRL+C in the Editor should only make a single copy and not multiple.

Bug Fixes

  • Fixed a major memory issue that would cause Snipping Tool++ to eventually use up to 500mb of RAM or more. This has been addressed, the tool shouldn't use more than 150mb and most of that is just allocated memory for the JVM that's not actually being used
  • Fixed the donate link on the donate button in the task trays right click menu
  • Fixed an issue that was causing the upload finished ding to not play at the correct time
  • Fixed a bounds issue when updating the overlay screenshot with middle mouse click
  • Fixed some grammar issues in the keysets help page

6.2.0 3rd Dec. 2014

New Features!

  • Magnifying Glass - When in the overlay you can enable/disable the magnifying glass with [Right Mouse Click]. When enabled, using the [Scrollwheel] will increase or decrease the level of zoom. NOTE: To change the overlay transparency, disable the Magnifying Glass with [Right Mouse Click] then use the [Scrollwheel] to increase/decrease transparency.
  • Master Hotkey - A master hotkey [PRINT SCREEN] can be used to bring up the upload overlay. This can be used instead of the default [CTL+SHIFT] modified hotkeys until we get customizable hotkeys in.

General Improvements

  • Download Progress Bar - Now when downloading your cloud history a progress bar will show the status of your history being downloaded as it comes in.
  • Images uploaded with the Multi-Image Uploader will now be added to your Anonymous Upload History meaning they will be downloaded with the rest of your anonymous uploads.
  • Temporary cloud history files are now deleted upon closing the Capture Viewer or exiting the program all together.

Misc.

  • A Donate button has been added to the [Right Click] tray menu.

6.1.2 24th Nov. 2014

New Features!

  • Cloud Based Upload History - No accounts or personal information required!

    To access this feature you need to register a unique keyset that will be used to sign your uploaded images so to speak. This keyset is simply two keys which can be any word or phrase you want, or you can generate a 26 character alpha numerical key. This keyset can be saved to a file which you can import if needed.

    You can view the images signed with your keyset using the [Utilities > Capture Viewer] on any computer that has a Snipping Tool++ client using your unique keyset.
  • Expect more uploading and statistic oriented features in future updates!

Bug Fixes!

  • Failure to connect to database was happening to every 6.1.1 release due to a bug in Java 7 when running from the .jar or .exe. This has been fixed in the 6.1.2 release.

6.1.1 22nd Nov. 2014

New Features!

  • Statistics foundation added.

    You can now create a unique identifier for yourself in Preferences > Stats/Keysets that can be used to anonymously sign your Imgur uploads. In future updates you will be able to look at your own personal upload statistics, have a retrievable upload history saved to the cloud, and even be able to download an entire album of images from your upload history to any computer you have a Snipping Tool++ client.

    No personal information or emails required! You stay completely anonymous!

6.1.0 2nd. Nov. 2014

New Features!

  • Upoad via FTP - Preferences > FTP to change your ftp settings
  • New Hotkeys - ALT+SHIFT+1 and ALT+SHIFT+2 can be used to upload snippets/screenshots respectively via FTP
  • Upload Quality - You can now change the quality of your uploads. Max quality is set by default.

Bug Fixes

  • Using the Editor hotkeys will no longer act as if you pressed them multiple times. This bug would cause the Editor to upload the same image multiple times or copy it multiple times
